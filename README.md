# 이음어린이집 캘린더

이음어린이집의 월별 당직표를 캘린더 형식으로 확인하는 Next.js 웹앱입니다. 당직은 `오전1`, `오전2`, `오후1`, `오후2`로 구분하며, 현재 데이터는 `2026년_7월_당직표.xlsx`를 기반으로 넣었습니다.

## 로컬 실행

```bash
npm run dev
```

브라우저에서 `http://localhost:3000`을 엽니다.

## 당직표 규칙

- 당직 슬롯은 `오전1`, `오전2`, `오후1`, `오후2` 네 가지입니다.
  - `오전1`: 7:30 ~ 16:30
  - `오전2`: 8:00 ~ 17:00
  - `오후1`: 9:00 ~ 18:30, 다음날 30분 일찍 퇴근
  - `오후2`: 9:30 ~ 19:30, 1시간 시간 외 수당 지급
- 2026년 7월 기준 `2026-07-01`에는 당직을 배정하지 않습니다.
- 당직 근무일은 `2026-07-02`부터 시작하며, 주말과 `2026-07-17` 제헌절을 제외한 21일입니다.
- 하루에는 서로 다른 선생님 4명이 각 슬롯에 1명씩 배정됩니다.
- 전체 배정은 84건이며, 인원별 총 횟수는 12회로 맞춥니다.
- 각 슬롯별 배정 횟수도 선생님별로 3회씩 맞춥니다.
- 같은 반을 맡는 선생님은 같은 날 오전 당직 묶음(`오전1`, `오전2`)이나 오후 당직 묶음(`오후1`, `오후2`)에 함께 배정하지 않습니다.
  - 포근반: 이혜빈, 정지혜, 홍여진
  - 다솜반: 김민경, 최옥희
  - 도담반: 어영경
  - 라온반: 봉은영
- `오후2`를 맡은 선생님은 다음 근무일의 `오전1` 또는 `오전2`에 배정하지 않습니다.
- `오후1`을 맡은 선생님은 다음 근무일에 0.5 조기퇴근을 하므로, 다음 근무일의 `오후1` 또는 `오후2`에 배정하지 않습니다.
  - 다음 근무일은 주말과 공휴일을 건너뛴 다음 당직일 기준입니다.
- 요일별 체감 균등성을 위해 각 선생님은 월~금 각 요일에 최소 2회, 최대 3회만 배정합니다.
- 같은 선생님이 같은 요일에 같은 슬롯을 반복해서 맡지 않도록 합니다.
- 완전히 동일한 하루 배정 패턴은 반복하지 않습니다.
- `npm run verify`는 위 규칙 중 배정 수, 슬롯 분포, 반 겹침, `오후2` 다음 근무일 오전 배정, `오후1` 다음 근무일 오후 배정, 요일 균등성, 반복 패턴을 검사합니다.

## Vercel 배포

```bash
npx vercel@latest
```

운영 배포는 다음 명령을 사용합니다.

```bash
npx vercel@latest --prod
```

이 프로젝트는 외부 런타임 의존성이 없는 정적 사이트라서 Vercel에서 별도 설정 없이 배포할 수 있습니다.
Next.js 프로젝트이므로 Vercel에서는 빌드 명령 `npm run build`를 실행한 뒤 Next 출력물을 자동으로 배포합니다.

## 다음 단계

- DB는 Supabase 우선 검토
  - Postgres 기반이라 당직표/일정표 데이터를 구조화하기 좋음
  - Auth와 Row Level Security를 이용해 권한이 있는 사용자만 편집 가능하게 만들기 쉬움
  - Realtime을 붙이면 여러 사용자가 보는 화면에 변경 사항을 바로 반영할 수 있음
- DB 구조는 당직과 일정을 모두 추가/수정할 수 있게 설계
  - `duties`: 날짜, 시간대(오전1/오전2/오후1/오후2), 담당자, 메모
  - `events`: 날짜, 제목, 시작/종료 시간, 설명, 색상, 공개 여부
  - `profiles`: 사용자 이름, 역할, 편집 권한
- 구글 캘린더처럼 이전/다음 달 이동
- 당직 추가, 수정, 삭제 기능
- 일정 추가, 수정, 삭제 기능
- 권한이 있는 사용자만 편집 가능한 관리 화면
